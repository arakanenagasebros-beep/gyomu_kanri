# 業務管理アプリ 手動確認チェックリスト

最終更新: 2026-04-01

## 事前準備

- `admin.html` と `staff.html` を強制再読み込みする
- 可能ならシークレットウィンドウか別プロファイルで確認する
- `config.js` の URL が意図した GAS を向いていることを確認する
- staff 用に 1 アカウント、admin 用に 1 アカウントを用意する

## Admin

### 1. ログインと初期遷移

- `admin.html#admin-login` を開く
- admin ログインが通る
- ログイン直後に `#admin-task-list` へ遷移する
- 画面上部タイトルが「業務管理」になっている
- 旧 `.admin-tabs` が表示されていない
- グローバルダッシュボードの導線が見える

### 2. 管理ナビ

- 業務管理 → 日報管理 → スタンプ管理 → 設定 → 月次確認 の順で遷移できる
- どの画面でもログアウト導線が残っている
- ダッシュボードの「今日の合言葉」が表示される

### 3. スタッフ編集オーバーレイ

- `設定` 画面のスタッフ一覧から `編集` を押す
- overlay が開く
- `ログインID / パスワード / スタッフ名 / 種別 / 時給` が埋まる
- overlay 外クリックと閉じるボタンで閉じられる
- 保存せず閉じても値が壊れない

### 4. 日報フロー

- `日報管理` 一覧が表示される
- 対象スタッフの日報詳細へ入れる
- 詳細から編集導線が動く
- レビュー操作が保存される
- 戻る導線で管理一覧へ戻れる

### 5. スタンプ修正フロー

- `スタンプ管理` で申請バッジが見える場合は対象スタッフへ入る
- 申請中のカレンダー差分が見える
- `承認` と `却下` ボタンが表示される
- 承認後または却下後に状態表示が崩れない

## Staff

### 1. ログインと初期遷移

- `staff.html#user-login` を開く
- 学生スタッフはログイン後 `#user-stamp` に行く
- 社員スタッフはログイン後 `#report-confirm` に行く
- staff ダッシュボードのナビが見える
- 旧ボタン群が非表示になっていて二重導線になっていない

### 2. パスワード変更

- staff ダッシュボードの `PW変更` を押す
- overlay が開く
- 空欄送信でバリデーションが出る
- 確認不一致でエラーが出る
- `Esc`、閉じるボタン、背景クリックで閉じられる
- 実変更は最後に 1 回だけ試す
- 変更後に再ログインできる

### 3. 日報入力

- `日報入力` に遷移できる
- 同日の日報一覧プレビューが表示される
- `日報一覧` に戻れる
- `業務一覧` に遷移できる

### 4. 日報一覧

- 月フィルタが動く
- 新規日報ボタンが動く
- 既存日報の編集導線が動く

### 5. スタンプ修正申請

- 学生スタッフで `スタンプ修正申請` ボタンが出る
- 編集モードへ入る
- 通常出勤 / 緊急出勤 の切替が動く
- 申請送信後に「申請中」表示へ変わる
- 管理側処理後に `承認済み` または `却下` 表示になる
- 表示クリア後に通常状態へ戻る

## 重点確認メモ

- admin / staff ともにログイン直後の体感が重くないか
- `syncPull()` を多めに呼ぶ画面で二重描画やチラつきがないか
- PW 変更後に admin のスタッフ編集 overlay でも `pw` が見えるか
- 画面名が `ホーム` ではなく `業務管理` / `スタンプ管理` に寄っているか
- 既存変更を壊していないことを優先し、保存系の試験は必要最小限にする
